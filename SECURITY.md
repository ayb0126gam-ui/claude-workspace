## ⚡ 골든 룰: 노출 의심 = 즉시 폐기

"노출됐나? 아닌가?" 고민하는 시간에 키를 폐기하고 새로 발급받는 게 가장 빠르고 안전합니다.
**키 재발급은 무료이고 5분이면 끝납니다.**

---

## 1️⃣ 어떤 키가 노출됐는지 확인

| 키 종류 | 어디서 쓰는지 | 노출 시 위험 |
|---------|-------------|------------|
| OpenRouter API 키 (`sk-or-v1-...`) | AI 모델 호출 | 내 돈으로 남이 AI 사용 (과금 폭탄) |
| Oracle SSH 키 (`oracle-server.key`) | 서버 접속 | 서버 통째로 탈취 |
| WordPress 비밀번호 | 블로그/사이트 관리 | 사이트 변조·삭제 |

---

## 2️⃣ 키별 대응 절차

### 🔑 OpenRouter API 키가 노출됐을 때

1. **폐기:** https://openrouter.ai/keys 접속 → 노출된 키 옆 **삭제(Delete)**
2. **재발급:** 같은 페이지에서 **Create Key** 로 새 키 발급
3. **교체:** `.env` 파일을 열어 `OPENROUTER_API_KEY=새키` 로 수정
4. **사용 이력 확인:** https://openrouter.ai/activity 에서 내가 안 쓴 호출이 있는지 확인
5. 이상한 사용 내역이 있으면 OpenRouter 고객지원에 문의

### 🖥️ Oracle 서버 SSH 키가 노출됐을 때

1. **즉시 서버 접속해서 새 키 등록 후 옛 키 제거:**
   - Oracle Cloud 콘솔 (https://cloud.oracle.com) 로그인
   - 새 SSH 키 쌍 생성 (Claude Code에게 "새 SSH 키 만들어줘"라고 요청)
   - 서버의 `~/.ssh/authorized_keys` 에서 옛 키 줄 삭제, 새 키 추가
2. **접속 이력 확인:** 서버에서 `last` 명령으로 모르는 접속 기록 확인
3. 모르는 접속이 있으면 → 서버 스냅샷 백업 후 인스턴스 재생성 고려
4. 로컬의 옛 키 파일(`~/.ssh/oracle-server.key`)은 새 키로 교체

### 📝 WordPress 비밀번호가 노출됐을 때

1. **즉시 비밀번호 변경:** 관리자 페이지 → 사용자 → 프로필 → 새 비밀번호
2. **활성 세션 모두 로그아웃:** 프로필 화면의 "다른 모든 곳에서 로그아웃" 클릭
3. **사용자 목록 확인:** 모르는 관리자 계정이 생겼는지 확인 → 있으면 삭제
4. **애플리케이션 비밀번호** 를 쓰고 있었다면 그것도 폐기 후 재발급
5. 플러그인/테마에 모르는 파일이 추가됐는지 확인

## 3️⃣ 교체 후 마무리 점검

- [ ] 새 키가 `.env` 에만 있고, 코드에 하드코딩 안 됐는지 확인
- [ ] `.env` 가 `.gitignore` 에 들어있는지 확인
- [ ] Git에 키가 커밋된 적 있다면 → **커밋 삭제만으론 부족!** 키 폐기가 정답 (이미 했으면 OK)
- [ ] 노출 경로 파악: 어디서 노출됐는지 (스크린샷? 코드 공유? Git 푸시?)
- [ ] `tasks/progress.md` 에 사건 기록 남기기

## 🛡️ 평소 예방 수칙

1. 키는 항상 `.env` 파일에만 보관 (절대 코드에 직접 쓰지 않기)
2. 화면 공유·스크린샷 전에 `.env` 파일 닫기
3. `.env`, `*.key`, `*.pem` 파일은 절대 `git add` 하지 않기
4. 키를 누군가에게 보여줘야 할 땐 마스킹: `sk-or-v1-***`
5. 의심되면 Claude Code에게: **"키 노출 의심돼"** 라고 말하면 이 매뉴얼대로 안내받을 수 있습니다
