Jerome의 Git 여정

포트폴리오를 만들면서 직접 경험한 Git 핵심 개념 시각화

커밋 그래프 — 이 프로젝트의 전체 타임라인
master (메인)
dark-mode 브랜치
design/cute-font 브랜치
revert (되돌리기)
HEAD (현재)
핵심 개념 6가지
📸
commit — 스냅샷 저장
git commit -m "메시지"
작업 내용을 타임라인에 기록. 한 번 커밋하면 언제든 그 시점으로 돌아올 수 있다. 사진 찍듯이 그 순간의 파일 상태를 저장하는 개념.
☁️
push — GitHub에 올리기
git push origin master
로컬(내 컴퓨터) 커밋을 원격(GitHub)으로 업로드. push 전까지는 내 컴퓨터에만 존재하고 GitHub에는 반영되지 않는다.
🌿
branch — 실험 공간
git checkout -b 이름
master를 건드리지 않고 새 기능을 실험하는 평행 우주. 이 프로젝트에서는 dark-mode 브랜치와 design/cute-font 브랜치를 만들었다.
🔀
merge — 합치기
git merge 브랜치명
브랜치 작업 내용을 master에 합치는 과정. 다른 줄을 수정했으면 자동 합산, 같은 줄이면 충돌 발생.
⚔️
conflict — 충돌
수동 해결 후 git commit
두 브랜치가 같은 줄을 다르게 수정했을 때 발생. Git이 어느 쪽이 맞는지 모르기 때문에 직접 선택해야 한다. 이 프로젝트에서 3곳 충돌 해결.
⏪
revert / checkout — 되돌리기
git revert HEAD
push한 커밋은 revert(취소 커밋 추가), push 전이면 reset 사용. checkout은 과거를 보기만 할 때. 기록은 항상 보존된다.
충돌 해결 — dark-mode 머지에서 겪은 3건

dark-mode 브랜치를 master에 머지할 때 3군데에서 충돌이 발생했다. 양쪽이 같은 줄을 다르게 수정했기 때문.

충돌 1 — CSS: 애니메이션 vs 토글 스타일
master
히어로 이름 바운스 애니메이션 CSS
→
dark-mode
다크모드 토글 스위치 CSS
✅ 해결: 둘 다 유지
충돌 2 — HTML: 프로필 이미지 vs 이모지 아바타
master
DiceBear 이미지 + Jerome Ko
→
dark-mode
이모지 아바타 + 김수민
✅ 해결: DiceBear 이미지 + Jerome Ko 선택
충돌 3 — JS: 이름 애니메이션 vs 토글 스크립트
master
이름 per-letter 바운스 JS
→
dark-mode
다크모드 토글 + localStorage JS
✅ 해결: 둘 다 유지
💡 핵심 원리: Git은 "공통 조상"과 비교해 판단한다. 한쪽만 바꿨으면 자동 채택. 양쪽이 같은 줄을 다르게 바꿨을 때만 충돌.
왜 cute-font 머지는 충돌이 없었을까?
공통 조상 (분기점)
히어로 색 = 초록(teal)
master
초록 그대로 ✓
이 줄 안 건드림
design/cute-font
핑크로 변경 🌸
이 줄 수정함
→ 한쪽(cute-font)만 변경했으므로 Git이 자동으로 핑크를 채택. 충돌 없음.
충돌 없는 경우
한쪽만 수정 → Git이 자동 채택
충돌 발생하는 경우
양쪽이 같은 줄을 다르게 수정 → 직접 선택
Git 명령어 치트시트

git status현재 변경사항 확인
git add -A모든 변경사항 스테이징
git commit -m "메시지"커밋 생성
git push origin masterGitHub에 업로드
git checkout -b 이름새 브랜치 생성 및 이동
git checkout mastermaster 브랜치로 이동
git merge 브랜치명브랜치를 현재 브랜치에 합치기
git checkout 해시 -- .특정 커밋 파일 상태로 복원

git checkout 해시과거 상태 보기만 (방문)
git branch브랜치 목록 보기
